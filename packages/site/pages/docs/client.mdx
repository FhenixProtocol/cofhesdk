# Client

This page covers the core SDK lifecycle:

1. Create a config (`createCofhesdkConfig`)
2. Create a client (`createCofhesdkClient`)
3. Create viem clients (either directly, or via `@cofhe/sdk/adapters`)
4. Connect + disconnect

## 1) Create a config

Pick the entrypoint that matches your runtime:

- Browser apps: `@cofhe/sdk/web`
- Node.js scripts/backends: `@cofhe/sdk/node`

The only required field is `supportedChains`.

```ts twoslash
// @noErrors
import { createCofhesdkConfig } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';

const config = createCofhesdkConfig({
  supportedChains: [chains.sepolia],

  // Optional tweaks
  // permitGeneration: 'ON_CONNECT',
  // defaultPermitExpiration: 60 * 60 * 24 * 30,
  // useWorkers: true,
});
```

## 2) Create a client

```ts twoslash
// @noErrors
import { createCofhesdkClient } from '@cofhe/sdk/web';
import type { CofhesdkConfig } from '@cofhe/sdk';

// config from the previous step
declare const config: CofhesdkConfig;

const cofhesdk = createCofhesdkClient(config);

cofhesdk.connected; // false
cofhesdk.connecting; // false
```

## 3) Create viem clients (adapters)

The SDK connects to CoFHE using viem clients:

- `PublicClient`: read-only chain access
- `WalletClient`: signing + sending transactions

If you already have viem clients, you can pass them directly to `cofhesdk.connect(publicClient, walletClient)`.

If you’re using another wallet/provider stack, `@cofhe/sdk/adapters` can convert it into viem-shaped clients.

### Ethers v6 adapter

```ts twoslash
// @noErrors
import { Ethers6Adapter } from '@cofhe/sdk/adapters';

// Your ethers instances
declare const provider: Parameters<typeof Ethers6Adapter>[0];
declare const signer: Parameters<typeof Ethers6Adapter>[1];

const { publicClient, walletClient } = await Ethers6Adapter(provider, signer);
```

### Ethers v5 adapter

```ts twoslash
// @noErrors
import { Ethers5Adapter } from '@cofhe/sdk/adapters';

// Your ethers instances
declare const provider: Parameters<typeof Ethers5Adapter>[0];
declare const signer: Parameters<typeof Ethers5Adapter>[1];

const { publicClient, walletClient } = await Ethers5Adapter(provider, signer);
```

### Wagmi adapter

If you’re using Wagmi, you already have viem clients; the adapter just validates and passes them through.

```ts twoslash
// @noErrors
import { WagmiAdapter } from '@cofhe/sdk/adapters';
import type { PublicClient, WalletClient } from 'viem';

declare const publicClient: PublicClient;
declare const walletClient: WalletClient;

const clients = await WagmiAdapter(walletClient, publicClient);
```

## 4) Connect

Connecting initializes the SDK’s in-memory connection state (`account`, `chainId`, and the viem clients).

```ts twoslash
// @noErrors
import { createCofhesdkClient } from '@cofhe/sdk/web';
import type { CofhesdkConfig } from '@cofhe/sdk';
import type { PublicClient, WalletClient } from 'viem';

declare const config: CofhesdkConfig;
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;

const cofhesdk = createCofhesdkClient(config);

await cofhesdk.connect(publicClient, walletClient);

cofhesdk.connected; // true
```

### Reconnect behavior

Calling `connect` again with the same clients is a no-op. Calling it with new clients replaces the connection state.

## 5) Disconnect

Disconnect clears only the in-memory connection state (clients/account/chainId) and marks the client as disconnected.

It does **not** delete persisted permits or stored FHE keys.

```ts twoslash
// @noErrors
declare const cofhesdk: { disconnect(): void; connected: boolean };

cofhesdk.disconnect();

cofhesdk.connected; // false
```
