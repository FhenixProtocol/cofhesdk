# Cofhe SDK

Build and integrate privacy-preserving dApps on Fhenix using fully homomorphic encryption (FHE).

- New here? Start with the [Quick Start](/quick-start).
- Want the full overview first? Read the [Introduction](/intro).

## Get started

::::steps

### 1) Install

If you’re building contracts and tests locally, install the SDK and the Hardhat plugin:

:::code-group

```bash [npm]
npm install @cofhe/sdk @cofhe/hardhat-plugin
```

```bash [pnpm]
pnpm add @cofhe/sdk @cofhe/hardhat-plugin
```

```bash [yarn]
yarn add @cofhe/sdk @cofhe/hardhat-plugin
```

:::

### 2) Run the end-to-end flow

Follow [Quick Start](/quick-start) to encrypt an input, store it on-chain, and decrypt it back in a test.

### 3) Go deeper

- Learn the core client API in [Client](/sdk/client)
- Encrypt contract inputs in [Encrypting Inputs](/sdk/encrypting-inputs)
- Show encrypted state in UIs in [Decrypting to View](/sdk/decrypt-to-view)
- Use on-chain verifiable decryption in [Decrypting to Transact](/sdk/decrypt-to-tx)
- Understand access control in [Permits](/sdk/permits)

::::

## What you can do with the SDK

| Goal                                        | Use                                              | Where to learn                                   |
| ------------------------------------------- | ------------------------------------------------ | ------------------------------------------------ |
| Encrypt inputs for contract calls           | `client.encryptInputs([...]).execute()`          | [/sdk/encrypting-inputs](/sdk/encrypting-inputs) |
| Decrypt handles for UI display              | `client.decryptForView(ctHash, utype).execute()` | [/sdk/decrypt-to-view](/sdk/decrypt-to-view)     |
| Decrypt + get a verifiable signature for tx | `client.decryptForTx(ctHash).execute()`          | [/sdk/decrypt-to-tx](/sdk/decrypt-to-tx)         |
| Create and manage decryption permits        | `client.permits.*`                               | [/sdk/permits](/sdk/permits)                     |
