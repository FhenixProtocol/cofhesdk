# Permits

A permit is a **wallet-signed approval** that lets your app decrypt confidential values.

At a high level, a permit looks like this:

```ts
type Permit = {
  hash: string; // Stable identifier used as the storage key
  name: string; // Human-readable label (not part of the signature)
  type: 'self' | 'sharing' | 'recipient'; // Self use, shareable, or imported-by-recipient

  issuer: `0x${string}`; // Address that grants the permission
  expiration: number; // Unix timestamp (seconds)
  recipient: `0x${string}`; // Who can use it (zero-address for self permits)

  validatorId: number; // Optional external validation id (0 disables)
  validatorContract: `0x${string}`; // Optional validator contract (zero-address disables)

  sealingPair: {
    publicKey: string; // Used by CoFHE to re-encrypt for you
    privateKey: string; // Used locally to unseal (keep secret)
  };

  issuerSignature: `0x${string}`; // EIP-712 signature by the issuer
  recipientSignature: `0x${string}`; // EIP-712 signature by the recipient (shared permits)
};
```

:::note
Under the hood, permits are signed using **EIP-712 typed data**.
:::

## When do you need a permit?

- **`decryptForView`**: always requires a permit (there’s no “no-permit” mode).
- **`decryptForTx`**: depends on the contract’s ACL policy for that `ctHash`.
  - If the policy allows anyone to decrypt, you can use `.withoutPermit()`.
  - If the policy restricts decryption, you must use `.withPermit(...)`.

## Prerequisites

1. Create and connect a client (see the [client page](./client.mdx)).

Permits are scoped to a **chainId + account**. The SDK uses the connected chain/account by default.

## Quickstart (create an active permit)

The quickest “just make it work” setup is to create (or reuse) a self permit and select it as the active permit:

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
// ---cut---

await client.connect(publicClient, walletClient);

// Creates a self permit if needed, stores it, and selects it as the active permit.
await client.permits.getOrCreateSelfPermit();
```

After this, builders that use permits can pick up the active permit automatically:

- `decryptForView(...).execute()` uses the active permit.
- `decryptForTx(...).withPermit().execute()` uses the active permit.

## Permit types

The SDK supports three permit shapes:

| Type        | Who signs                                 | Intended use                                                  |
| ----------- | ----------------------------------------- | ------------------------------------------------------------- |
| `self`      | issuer only                               | Your own permit for decrypting data as yourself (most common) |
| `sharing`   | issuer only                               | A shareable “offer” created by issuer and sent to a recipient |
| `recipient` | recipient (and includes issuer signature) | The imported shared permit after the recipient signs it       |

Notes:

- A permit includes a **sealing keypair**.
  - The public key is used by CoFHE to re-encrypt for you.
  - The private key stays client-side and is used to unseal the payload.
- Permit `expiration` is a unix timestamp in **seconds**.
  - If you don’t provide it, the default is currently **7 days from creation**.

## Creating a self permit

Create a self permit for the connected wallet and store it as the active permit:

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
// ---cut---

await client.connect(publicClient, walletClient);

const permit = await client.permits.createSelf({
  issuer: walletClient.account!.address,
  name: 'My self permit',
});

permit.type;
permit.hash;
```

If you want the SDK to reuse the active self permit (instead of always creating a new one), prefer `getOrCreateSelfPermit()`:

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
// ---cut---

await client.connect(publicClient, walletClient);

// Returns the active self permit if present, otherwise creates it.
const permit = await client.permits.getOrCreateSelfPermit();
permit.type;
```

## Sharing permits

Sharing permits are a two-step flow:

1. **Issuer** creates a `sharing` permit and shares an exported JSON blob with the recipient.
2. **Recipient** imports it and signs it, producing a `recipient` permit that they can use.

### 1) Issuer creates + exports

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import { PermitUtils } from '@cofhe/sdk/permits';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
declare const recipient: `0x${string}`;
// ---cut---

await client.connect(publicClient, walletClient);

const sharingPermit = await client.permits.createSharing({
  issuer: walletClient.account!.address,
  recipient,
  name: 'Share with recipient',
});

// Export a shareable payload (does not include your sealing private key).
const exported = PermitUtils.export(sharingPermit);

exported;
```

:::note
Do not share `client.permits.serialize(permit)` output with other users.
Serialization is meant for _local persistence_ and includes the sealing private key.
For sharing, use `PermitUtils.export(...)`.
:::

### 2) Recipient imports + signs

The recipient imports the JSON (string or object) and signs it with their wallet. The resulting permit is stored and set as active for the recipient’s `chainId + account`.

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
declare const exported: string;
// ---cut---

await client.connect(publicClient, walletClient);

const recipientPermit = await client.permits.importShared(exported);

recipientPermit.type;
recipientPermit.hash;
```

## Active permit management

The permits store tracks:

- all stored permits per `chainId + account`, and
- an **active permit hash** per `chainId + account`.

Creating or importing a permit via `client.permits.*` will store it and select it as active.

### List stored permits

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
// ---cut---

await client.connect(publicClient, walletClient);

const permits = client.permits.getPermits();
Object.keys(permits);
```

### Read/select the active permit

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
declare const somePermitHash: string;
// ---cut---

await client.connect(publicClient, walletClient);

const active = client.permits.getActivePermit();
active?.hash;

client.permits.selectActivePermit(somePermitHash);
```

### Removing permits

```ts twoslash
import { createCofheConfig, createCofheClient } from '@cofhe/sdk/web';
import { chains } from '@cofhe/sdk/chains';
import type { PublicClient, WalletClient } from 'viem';

const config = createCofheConfig({ supportedChains: [chains.sepolia] });
const client = createCofheClient(config);
declare const publicClient: PublicClient;
declare const walletClient: WalletClient;
declare const permitHash: string;
// ---cut---

await client.connect(publicClient, walletClient);

client.permits.removePermit(permitHash);
client.permits.removeActivePermit();
```

## Persistence & security

- The SDK keeps permits in a persisted store keyed by `chainId + account`.
- In web/react environments, this store is persisted in `localStorage` under the key `cofhesdk-permits`.
- A stored permit includes your **sealing private key**. Treat it like a secret.
  - Do not send serialized permits to other users.
  - To share, export a sharing permit via `PermitUtils.export(...)`.

## Advanced: external validators (optional)

A permit can optionally reference an external on-chain validator via:

- `validatorContract` (address)
- `validatorId` (number)

If provided, the ACL can query that validator contract to decide whether the permit is valid.

You can also ask the SDK to simulate the ACL validity check:

```ts twoslash
import { PermitUtils } from '@cofhe/sdk/permits';
import type { Permit } from '@cofhe/sdk/permits';
import type { PublicClient } from 'viem';

declare const permit: Permit;
declare const publicClient: PublicClient;
// ---cut---

const ok = await PermitUtils.checkValidityOnChain(permit, publicClient);
ok;
```
