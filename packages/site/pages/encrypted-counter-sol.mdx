# EncryptedCounter.sol

A minimal FHE-enabled contract that showcases the full range of SDK functions: encrypting inputs, reading encrypted state, managing permits, and decrypting for both view and transaction.

### Encrypting inputs

Confidential values must be encrypted before being used as a parameter, else they will be revealed during a contract call. `setCount` accepts an encrypted input (`InEuint32`) prepared by the client. The contract verifies it and stores it as the new count.

### View encrypted data

To view encrypted data in a client, it must be decrypted using `decryptForView`. This function uses the `ctHash` representing the encrypted value, which in this contract can be fetched directly via the state variable or with the `getCount` function. Both options return a `bytes32`, treated as a string in typescript.

### Decrypt for a transaction

Some contract flows require an encrypted value to be available as plaintext on-chain. Cofhesdk exposes the function `decryptForTx` to allow this reveal. `decryptForTx` returns the decrypted value as plaintext and an on-chain verifiable signature.

`revealCount` accepts the plaintext value and signature generated by `decryptForTx` and stores the plaintext value as a state variable.
